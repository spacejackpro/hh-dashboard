# HH Dashboard — автоотклики на hh.ru

Локальное приложение для автоматической рассылки откликов на hh.ru.
Внутри работает утилита [hh-applicant-tool](https://github.com/s3rgeym/hh-applicant-tool),
а дашборд — удобная оболочка над ней. Всё работает только на этом компьютере,
наружу уходят только запросы к hh.ru.

---

## Как запустить

**Windows:** дважды кликни **`dashboard-hidden.vbs`** — приложение запустится
**без чёрного окна**, откроется только вкладка браузера. Чтобы выключить,
нажми «Выключить приложение» в левом нижнем углу самого приложения.

Если что-то работает не так и надо увидеть, что происходит, запусти
**`dashboard.cmd`** — то же самое, но с видимым окном сервера. При скрытом
запуске всё, что было бы в окне, пишется в файл `server.log` в этой папке.

**macOS:** дважды кликни **`dashboard.command`**. Если Mac скажет, что файл
«от неустановленного разработчика», кликни по нему правой кнопкой →
«Открыть» → «Открыть» (это нужно только один раз для каждого `.command`-файла).

Что произойдёт: откроется окно терминала (это сервер — его просто не
закрывай, можно свернуть) и вкладка браузера с адресом
`http://127.0.0.1:8517` — это и есть приложение. Чтобы выключить — закрой
окно терминала.

При самом первом запуске на новом компьютере лаунчер сам скачает и установит
всё необходимое (нужен интернет, займёт несколько минут) — отдельно ничего
ставить не надо. На macOS нужен установленный Python 3.11+ (обычно уже есть;
если нет — лаунчер подскажет, откуда взять).

## Вкладка «Обзор»

Первый экран. Что на нём:

- **«N / 200 откликов сегодня»** — сколько откликов уже отправлено сегодня.
  200 — это лимит hh.ru на сутки, больше физически не даст отправить никто.
- **«Откликов всего»** — за всё время работы.
- **«Пропущено вакансий»** — сколько вакансий утилита посмотрела, но решила
  не откликаться (почему — на вкладке «Пропущенные»).
- **«Работодателей в базе»** — размер накопленной базы компаний.
- **Таблица «Резюме»** — все твои резюме с количеством просмотров.
  Клик по названию откроет резюме на hh.ru.

Если вместо этого висит красная плашка **«Ты не авторизован»** — см. раздел
«Авторизация» ниже.

## Вкладка «Отклики»

Список всех отправленных откликов: вакансия (кликабельна), работодатель,
зарплата, статус и дата. Статусы:

- `response` — отклик отправлен, ответа пока нет;
- `invitation` — тебя пригласили! Иди на hh.ru и отвечай;
- `discard` — отказ.

## Вкладка «Пропущенные»

Вакансии, которые попались в поиске, но отклик не был отправлен, с причиной:
уже откликался, вакансия с тестом, работодатель в чёрном списке и т.п.
Полезно проверять: вдруг что-то хорошее пропущено зря.

## Вкладка «Запуск»

Главная рабочая вкладка. Поля формы «Рассылка откликов»:

| Поле | Что делает |
|---|---|
| **Каким резюме откликаться** | Выбор из твоих резюме. Если не выбирать — возьмёт первое из списка |
| **Ключевые слова** | Список сохранённых слов. Отметь нужные — вакансия подойдёт, если совпало **любое** из них («CRM-аналитик» ИЛИ «Битрикс24»). Слова сохраняются между запусками, добавить — поле ввода и «Добавить», удалить — крестик |
| **Где искать эти слова** | Ничего не отмечено — hh.ru ищет везде. Можно сузить: только в названии вакансии, в названии компании, в описании |
| **Города и страны** | Начни вводить название — появятся подсказки (город, регион, страна целиком). Можно выбрать несколько. Пусто — искать везде |
| **Мин. зарплата** | Отсеет вакансии с зарплатой ниже. Пусто — не фильтровать |
| **Максимум откликов за запуск** | Стоп-кран: отправив столько, рассылка остановится |
| **Сопроводительное письмо** | Шаблон письма работодателю. Подстановки: `%(vacancy_name)s` — вакансия, `%(employer_name)s` — компания, `%(first_name)s` — твоё имя, `%(resume_title)s` — резюме; `{Вариант 1\|Вариант 2}` — случайный выбор. Сохраняется автоматически |
| **Отправлять письмо всем** | Без галочки письмо уходит только там, где работодатель его требует |
| **Только с указанной зарплатой** | Пропускать вакансии, где зарплата не написана |
| **Пропускать вакансии с тестами** | Не трогать вакансии, где работодатель требует пройти тест |
| **Пробный запуск** ✅ | ВАЖНО: пока включено — реальные отклики НЕ отправляются, в логе просто видно, кого бы он выбрал |

**Что будет, если нажать «Запустить рассылку»:** внизу в «Логе» побегут
строки — какие вакансии нашлись, куда откликнулся (или откликнулся бы,
если пробный запуск), что пропустил и почему. Между откликами утилита
делает паузы — hh.ru разрешает максимум 5 откликов в минуту, так что
10 откликов занимают пару минут. Это нормально.

**Рекомендуемый порядок:** сначала запуск с галочкой «Пробный» → смотришь
лог → если вакансии подходящие, снимаешь галочку и запускаешь по-настоящему.

Кнопки справа:

- **«Поднять все резюме»** — то же, что кнопка «Поднять в поиске» на hh.ru
  (за которую hh берёт деньги): резюме поднимаются выше в выдаче у
  работодателей. Можно жать раз в 4 часа, чаще hh.ru не даст.
- **«Проверить авторизацию»** — покажет в логе, под каким аккаунтом работаем.
- **«Остановить операцию»** — прервать запущенную рассылку. Уже отправленные
  отклики не отменяются.
- **«Выйти из аккаунта»** — отзывает доступ и стирает токен с этого
  компьютера. Пригодится, чтобы зайти под другим аккаунтом.

## Авторизация

Токен (пропуск на hh.ru) живёт **2 недели**, потом на «Обзоре» появится
плашка с кнопкой **«Войти в hh.ru»**. Жмёшь — откроется окно браузера со
входом hh.ru, входишь как обычно (пароль или код из SMS), окно закроется
само, дашборд обновится. Пароль нигде не сохраняется — только на hh.ru.

### Если на почте два аккаунта: соискатель и работодатель

hh.ru разрешает держать на одной почте оба аккаунта, и при входе часто
подсовывает работодателя. Приложению нужен **соискатель** (только он может
откликаться на вакансии).

Признак проблемы: в левом нижнем углу «⚠» вместо «✓», а на «Обзоре» —
жёлтая плашка «Ты вошёл как работодатель».

Решение: нажми **«Войти как соискатель»** (или ссылку «У меня на этой почте
два аккаунта» в плашке входа). Откроется окно hh.ru, и приложение будет
ждать: войди, переключись на аккаунт соискателя (меню профиля в правом
верхнем углу), убедись, что видишь свои резюме, — и нажми в приложении
«Я в аккаунте соискателя — продолжить».

## Безопасность и лимиты

- Всё работает локально; пароль не хранится; токен и база лежат в
  `C:\Users\<имя>\AppData\Roaming\hh-applicant-tool\`.
- Для hh.ru запросы выглядят как от официального мобильного приложения.
- Лимиты hh.ru: **200 откликов в сутки**, не чаще 5 в минуту — утилита
  соблюдает их сама.

## Если что-то не работает

1. Закрой чёрное окно сервера и запусти `dashboard.cmd` заново.
2. Посмотри последние строки в «Логе» на вкладке «Запуск» — обычно причина
   написана по-русски (чаще всего «Требуется авторизация» → войди заново).
3. Если совсем беда — подробный лог утилиты лежит в
   `AppData\Roaming\hh-applicant-tool\log.txt`.

## Обновления

При запуске приложение само проверяет, не вышла ли новая версия. Если вышла —
на «Обзоре» появится жёлтая плашка с кнопкой **«Обновить сейчас»**: жмёшь,
приложение само скачивает свежий код, перезапускается и обновляет движок.
Твои данные и авторизация при этом не трогаются.

Запасной способ (если кнопка не сработала): закрой приложение (окно
сервера) и запусти **`update.cmd`** (Windows) или **`update.command`**
(macOS) — результат тот же.

## Как поделиться с другом

Вариант 1 (проще всего): скинь другу ссылку на GitHub-репозиторий — там
зелёная кнопка «Code» → «Download ZIP».

Вариант 2: заархивируй эту папку, **кроме папки `.venv`** (она огромная и на
чужом компьютере всё равно не заработает — внутри пути под этот ПК).

Дальше одинаково: друг распаковывает архив куда угодно и запускает
**`dashboard.cmd`** (Windows) или **`dashboard.command`** (macOS) — при
первом запуске всё установится само, откроется приложение, друг жмёт «Войти
в hh.ru» и входит в СВОЙ аккаунт. Одно и то же приложение работает на обеих
системах, отличаются только файлы запуска. Твои токены и история в архив не
попадают — они лежат в системной папке, вне этой папки. Обновления друг
получает той же кнопкой `update.cmd` / `update.command`.

Никогда не пересылай никому папку
`C:\Users\<имя>\AppData\Roaming\hh-applicant-tool` — там твой токен,
это равносильно передаче доступа к аккаунту.

## Что лежит в папке

| Файл/папка | Что это |
|---|---|
| `dashboard-hidden.vbs` | Запуск без окна (Windows, основной способ) |
| `dashboard.cmd` / `dashboard.command` | Запуск с видимым окном сервера (Windows / macOS). При первом запуске сам всё установит |
| `server.log` | Что происходило при скрытом запуске — сюда смотреть, если что-то сломалось |
| `update.cmd` / `update.command` | Обновление до свежей версии с GitHub |
| `setup.cmd` / `setup.command` | Установка (лаунчер вызывает его сам, отдельно обычно не нужен) |
| `requirements.txt` | Список библиотек для установки |
| `VERSION` | Номер текущей версии (по нему работает проверка обновлений) |
| `dashboard/` | Код дашборда (оболочки), общий для Windows и macOS |
| `.venv/` | Python, утилита hh-applicant-tool и все библиотеки — изолированно от системы. В архив не класть |
